無料でブログを立ち上げたいのでGitHub pagesとHugoを組み合わせて構築してみた

Feb 11, 2024 · 6 mins read
GitHub Hugo 技術

はじめに

この記事はある程度知識のある方向けとなっております。

ブログを構築したい

いままでいくつかのサービスに手を出しては面倒になってやめてを繰り返してきました。 そこでこれなら持続可能なアウトプットを出せるのでは？！となったので、構築してみました。

選定

私の要求は次のとおりでした。

  • 手元のvscodeを使ってmarkdownを書きたい
  • gitで管理したい
  • 無料ブログじゃないやつ

markdownを使いたい理由としては、万が一お引越しするとかになっても楽そうだなと。 また、gitで管理したい理由としては、変更履歴とかrevertとか気軽にできるなと思ったからです。 無料ブログを避けたい理由としては、私ではアンコントローラブルなことが起きると嫌だなと思ったからです。

じゃあGitHubもそうじゃんといえばそうですが、この場合は最悪とこかのGCSなりS3なりにお引越しして、CDNでも挟めばいいかｗ という気持ちでいるのでまあいいかなと思っています。

また、vscodeを使って記事を書くことを想定していますので、copilotによる補完とかが地味に便利なので活用しています。 あとオフラインでも文字を書けるのはでかいと思っています。あとフォーマットも適当にやってくれるのも嬉しいです。

選定結果

その結果、この体制で進めることにしました。

  • GitHub pagesにホスティング
  • Hugoで静的生成

git pushなりすればworkflowだかが動いてGitHub pagesにホスティングって感じにしました。

構築手順

hugo の準備

まずは適当にGitHubでリポジトリを作りましょう。

次はお手元にHugoを入れましょう。

Macであるなら

brew install hugo

で入りますのでサクッと入れましょう。

hugo new content なんたらかんたら

すればテンプレートが生成されます。

GitHub actions の準備

DNS をいじる

今のままではGitHub pagesのデフォルトのアドレスでしか見ることができません。 このままでもいいですが、ドメインちゃんとしたいな〜〜って人はドメインを取ってDNSをいじれば、 好きなアドレスにできます！！

DNSなんぞや？？の人はとりあえずインターネット上の住所を教えてくれる交番だと思ってください。

このサイトのように blog.travel-world.infoにアクセスしてもらいたい場合は、 DNSの人にblog.travel-world.infoは``にあるでを伝えておけば、 あとはうまいことアクセスできるようになります。

完成

あとは自由気ままに適当に書いていくだけです。
